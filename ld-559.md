## 项目概述

**宠物医疗与保险管理平台（PetCare+）** — 面向宠物主人、兽医和管理员的一站式宠物健康管理平台。整合宠物档案、就诊记录、疫苗接种计划、保险保单管理，支持 RBAC 权限控制和提醒通知系统，实现宠物全生命周期的健康数据追踪。

**项目类型：全栈Web应用**

## 项目主要需求

### 一、核心实体

1. **宠物 Pet**：字段包括 id、ownerId、name、species（猫/狗/兔/鸟/其他）、breed、gender、birthDate、weight、microchipNo、avatar、allergies、medicalHistory。业务动作：添加宠物、编辑宠物信息、查看完整病历。关系：宠物属于宠物主人（PetOwner），关联多条就诊记录、疫苗记录和保单。此实体贯穿全栈：DB → Prisma schema → PrismaService → PetsController → REST API → React Query → PetCard/PetDetail 组件。

2. **就诊记录 MedicalRecord**：字段包括 id、petId、vetId、clinicId、visitDate、type（常规/急诊/疫苗/手术/体检）、diagnosis、treatment、prescription、cost、nextVisitDate、attachments[]。关系：就诊关联宠物 + 兽医 + 诊所。业务动作：创建就诊记录、编辑诊断信息、查看/打印处方。此实体贯穿全栈：DB → Prisma schema → PrismaService → MedicalController → REST API → React Query → MedicalRecordList/MedicalRecordDetail 组件。

3. **疫苗记录 VaccineRecord**：字段包括 id、petId、vaccineName、batchNo、administeredDate、nextDueDate、vetId、status（已接种/待接种/已过期）。业务动作：记录接种信息、触发续种提醒、更新接种状态。此实体贯穿全栈：DB → Prisma schema → PrismaService → VaccineController → REST API → React Query → VaccineCalendar/VaccineList 组件。

4. **保险保单 InsurancePolicy**：字段包括 id、petId、provider、planType（基础/标准/高级）、premium、coverage、startDate、endDate、status（生效/待续/已过期/理赔中）。业务动作：投保新保单、续保操作、提交理赔、查看保单详情。此实体贯穿全栈：DB → Prisma schema → PrismaService → InsuranceController → REST API → React Query → InsuranceCard/ClaimForm 组件。

### 二、核心页面（≥ 4 个）

1. **`/pets` 我的宠物**：宠物卡片网格列表，每张卡片展示头像、名字、品种、年龄，支持搜索和按物种筛选，提供"添加宠物"入口。消费 Pet 实体数据。

2. **`/pets/:id` 宠物详情**：Tab 布局包含基本信息卡、就诊记录时间线、疫苗接种日历视图、关联保单列表。消费 Pet + MedicalRecord + VaccineRecord + InsurancePolicy 四个实体。

3. **`/medical` 就诊管理**：就诊记录表格列表（支持按宠物、类型、日期筛选），点击展开处方详情侧边栏，底部费用统计图表（ECharts 柱状图）。消费 MedicalRecord 实体。

4. **`/vaccines` 疫苗管理**：日历视图展示接种计划，待接种提醒列表（高亮过期项），已接种记录表格，一键标记接种完成。消费 VaccineRecord 实体。

5. **`/insurance` 保险中心**：保单卡片列表，理赔进度流程图，年度保费/理赔费用分析饼图。消费 InsurancePolicy 实体。

### 三、共用组件

- `PetAvatar`：宠物头像组件，支持图片/默认头像 fallback（被宠物列表页、就诊记录页、疫苗管理页共同使用）
- `VaccineCalendar`：疫苗日历组件，按月展示接种计划（被宠物详情页、疫苗管理页共同使用）
- `StatusBadge`：状态徽章组件，根据 status 类型显示不同颜色标签（被就诊记录页、疫苗管理页、保险中心页共同使用）

### 四、横切关注点

1. **认证授权 + RBAC**：支持三种角色 PetOwner / Vet / Admin，JWT 认证 + 角色中间件。PetOwner 只能查看自己的宠物及其关联数据，Vet 可查看分配给自己的就诊记录，Admin 拥有全部权限。此关注点触达数据库（users 表 + 角色字段）、后端中间件（auth.guard.ts + roles.guard.ts）、前端路由守卫、按钮权限指令、数据查询范围过滤。

2. **提醒通知系统**：疫苗到期前 7 天提醒、保险续保到期前 30 天提醒、复诊日期提醒。后端定时任务扫描待提醒记录，生成通知存入数据库，前端通知铃铛组件拉取未读通知。此关注点触达数据库（notifications 表）、后端定时任务模块（schedule 模块）、通知 Service、前端 NotificationBell 组件。

## 代码架构要求（强制执行）

### 枚举必须单独维护为常量文件

后端在 `backend/src/constants/enums.ts` 中集中定义所有枚举，前端在 `frontend/src/constants/enums.ts` 中维护一份完全一致的枚举定义。业务代码中**严禁硬编码**枚举值字符串。README 中必须列出所有枚举的定义位置和使用位置对照表（前端 + 后端分别列出）。

**共享枚举定义**：
- `PetSpecies`：CAT / DOG / RABBIT / BIRD / OTHER
- `VisitType`：ROUTINE / EMERGENCY / VACCINE / SURGERY / CHECKUP
- `VaccineStatus`：COMPLETED / PENDING / OVERDUE
- `InsuranceStatus`：ACTIVE / PENDING_RENEWAL / EXPIRED / CLAIMING
- `PolicyType`：BASIC / STANDARD / PREMIUM
- `Gender`：MALE / FEMALE

### 全局异常处理

- **后端统一异常中间件**：在 `backend/src/middleware/error-handler.ts` 中实现全局异常过滤器，拦截所有未捕获异常，统一返回格式 `{ code, message, data }`，区分业务异常（BusinessException）和系统异常，禁止 Controller 中直接 `res.status(500).send('error')`。
- **前端请求拦截器**：在 `frontend/src/utils/request.ts` 中封装 Axios 实例，配置响应拦截器拦截 HTTP 错误和业务错误码，统一调用 Ant Design 的 `message.error()` 展示错误提示，401 时自动跳转登录页，禁止各页面自行 catch 后直接弹窗。

### 操作日志

关键写操作（创建就诊记录、疫苗接种、投保、理赔、添加宠物）通过审计中间件记录操作日志。后端在 `backend/src/middleware/audit-log.ts` 中实现，自动记录请求路径、请求方法、操作人 ID、请求体摘要、响应状态码、操作时间，日志写入数据库 audit_logs 表。在 Controller 方法上使用 `@AuditLog('描述')` 装饰器标记需要记录的操作。

### 代码严禁合并职责到单一文件

- 每个功能模块必须拆分为独立文件。例如：宠物模块拆分为 `pet.controller.ts`、`pet.service.ts`、`pet.repository.ts`、`pet.routes.ts`、`pet.validator.ts`、`pet.dto.ts`。
- 前端组件按功能拆分，每个页面拆分为多个子组件，单个文件不超过 300 行。
- 后端按实体/功能拆分为 controller、service、repository、middleware、validator、routes、dto 等独立文件。
- Prisma schema 按领域分文件组织。

### 核心实体贯穿全栈

每个核心实体必须完整贯穿：**Prisma Schema → PrismaService → Repository → Service → Controller → Route → 前端 API 层 → React Query Hook → Store/State → 组件**。禁止跳过任何环节，例如禁止在 Controller 中直接调用 Prisma Client，必须通过 Service 层。

## 部署要求

项目为前后端分离，支持 Docker Compose 一键部署，且在**任意目录名（包括中文目录名）**下都能正常启动。

需要提供以下文件：
- 根目录 `docker-compose.yml`
- 根目录 `.env`：包含 `COMPOSE_PROJECT_NAME=ldpetcare`
- `frontend/Dockerfile`：多阶段构建（先 build 再用 Nginx 托管）
- `backend/Dockerfile`：Node.js 构建运行镜像
- `frontend/nginx.conf`：Nginx 配置，SPA 路由 + API 反向代理 + WebSocket 支持
- `.env.example`

docker-compose.yml 要求：
- **不要写 `version:` 字段**
- 顶层声明 `name: ldpetcare`
- 所有服务使用 `container_name`，带 `${COMPOSE_PROJECT_NAME:-ldpetcare}` 前缀
- PostgreSQL 数据使用命名卷持久化
- 前端通过 Nginx 反向代理后端 API（`/api` 转发到 `backend:38506`）
- 数据库配置 `healthcheck`，后端 `depends_on` 数据库 healthy
- 前端 `depends_on` 后端
- 暴露端口：前端 `38408:80`，后端 `38506:38506`
- 支持 `docker compose up -d` 一条命令启动

`.env` / `.env.example` 必须包含的键：
- `COMPOSE_PROJECT_NAME=ldpetcare`
- `DB_NAME` / `DB_USER` / `DB_PASSWORD` / `DB_ROOT_PASSWORD`
- `JWT_SECRET`
- `FRONTEND_PORT=38408`
- `BACKEND_PORT=38506`
- `DATABASE_URL`（Prisma 连接字符串）

Nginx 反向代理要求（`frontend/nginx.conf`）：
- `location /api/` 反向代理到 `http://backend:38506/api/`
- 配置 `try_files $uri $uri/ /index.html;` 支持 SPA 路由
- WebSocket 场景配置 `Upgrade` / `Connection` 头

自检清单：
1. 在中文目录名下执行 `docker compose config --quiet` 不报错
2. 启动后 `docker compose ps` 显示全部容器 healthy
3. 浏览器访问前端端口能看到登录页
4. API 请求 `/api/v1/pets` 返回正常

## docker-compose.yml 参考骨架

```yaml
name: ldpetcare

services:
  db:
    image: postgres:16-alpine
    container_name: ${COMPOSE_PROJECT_NAME:-ldpetcare}-db
    restart: unless-stopped
    environment:
      POSTGRES_DB: ${DB_NAME}
      POSTGRES_USER: ${DB_USER}
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    volumes:
      - db_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${DB_USER} -d ${DB_NAME}"]
      interval: 5s
      timeout: 5s
      retries: 20

  backend:
    build: ./backend
    container_name: ${COMPOSE_PROJECT_NAME:-ldpetcare}-backend
    restart: unless-stopped
    environment:
      DATABASE_URL: postgresql://${DB_USER}:${DB_PASSWORD}@db:5432/${DB_NAME}
      JWT_SECRET: ${JWT_SECRET}
      PORT: 38506
    ports:
      - "${BACKEND_PORT:-38506}:38506"
    depends_on:
      db:
        condition: service_healthy

  frontend:
    build: ./frontend
    container_name: ${COMPOSE_PROJECT_NAME:-ldpetcare}-frontend
    restart: unless-stopped
    ports:
      - "${FRONTEND_PORT:-38408}:80"
    depends_on:
      - backend

volumes:
  db_data:
```
